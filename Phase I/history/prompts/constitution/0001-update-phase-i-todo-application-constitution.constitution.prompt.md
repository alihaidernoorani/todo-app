## Mission

Build an in-memory, console-based Todo application in Python using **Spec-Driven Development (SDD)**, with all code generated exclusively through **Claude Code via Spec-Kit Plus**.

## Core Principles

- **Spec-Driven Development First**
  Constitution ‚Üí Specification ‚Üí Plan ‚Üí Tasks ‚Üí Implementation
  No code without an approved spec.

- **Zero Manual Coding**
  All code must be generated by Claude Code. Manual edits are forbidden.

- **Simplicity Before Scale**
  Focus on correctness, clarity, and extensibility‚Äînot persistence or UI polish.

- **Deterministic Behavior**
  Same input must always produce the same output.

---

## Phase I Scope

### Included
- In-memory storage only
- Python console I/O
- Single-user usage
- Clear separation of logic and UI
- Modular, testable design

### Excluded
- File I/O or databases
- APIs, web UI, or networking
- AI / LLM integrations
- Authentication
- Background processes
- Cloud or container tooling

---

## Technical Standards

- **Language:** Python 3.11+
- **Data Model:** Plain Python objects (dataclasses preferred)
- **Dependencies:** Standard library only
- **Error Handling:** Clear, human-readable messages
- **Input Validation:** Required for all user input

---

## Specification Rules

- Each feature spec must define:
  - Intent
  - Inputs
  - Outputs
  - Edge cases
  - Failure conditions

- Specs must be Markdown, deterministic, and implementation-agnostic.
- Specs may be refined until Claude Code produces correct behavior.

---

## Quality Bar

- No speculative abstractions
- Clear separation of concerns:
  - Task model
  - Business logic
  - Console interaction

- Phase III (AI agent control)
- Phase IV (Containerization)
